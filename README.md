# Student Information REST-APIs (Backend)

This is a **backend application** that manages student records using a **structured layered architecture** including **Controller, Service, and Repository layers**.  
The project demonstrates how to organize backend code following **best practices** for maintainability and separation of concerns.

---

## Key Features
- Manage student records: add, update, delete, and fetch details  
- Structured layered architecture:
  - **Controller** – handles API requests  
  - **Service** – contains business logic  
  - **Repository** – interacts with the database  
- Clean separation of concerns for easier maintenance and scalability
